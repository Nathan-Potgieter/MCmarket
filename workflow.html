Types of Markets

The first step is to think about what kind of market it is that you want to simulate. So lets start with the looking at some correlation matrices.

ad hoc/ constructed correlation matricies

  • The function gen_corr allows users to easily generate ad hoc correlation matrices with a set number of clusters and up to 4 layers.
pacman::p_load(MCmarket, tidyverse, ggcorrplot)

# First look at the diagonal correlation matrix
corr_1 <- diag(50)

# Correlation matrix with no clusters
corr_2 <- gen_corr(D = 50, clusters = "none")
corr_2 %>% ggcorrplot(title = "No Clusters")

# Correlation matrix with 5 clusters
corr_3 <- gen_corr(D = 50, clusters = "non-overlapping", num_clusters = 5)
corr_3 %>% ggcorrplot(hc.order = TRUE, title = "Five Clusters")

corr_4 <- gen_corr(D = 50, clusters = "overlapping", num_clusters = c(10,5,2))

Emperical

If the ad hoc matrices don’t seem realistic enough then one can use empirical data to estimate a correlation matrix of their own.

MCmarket also comes with a data set of 30 correlation matrices that were each estimated from a random subset of 50 S&P500 stocks over a random time slice of 252 days between 1 January 2000 and 1 January 2020. These correlation matrices are labeled as ‘stressed’ if they were estimated on a market where an equi-weighted basket of stocks had a Sharpe ratio below -0.5, ‘rally’ if the equi-weighted basket of stocks had a Sharpe ratio above 2 and ‘normal’ if the equi-weighted basket of stocks had a Sharpe in-between -0.5 and 2. See MCmarket::corr_mats

Multivariate Distributions

The 2nd step in deciding what kind of market to simulate is deciding on the multivariate distribution. MCmarket provides functionality to select between the multivariate normal, t and Clayton copula distribution. However, note that when using the Clayton copula option the simulated markets no longer adhere to the input correlation matrix.

Univariate/Marginal Distributions

The next step is to decide on the return distributions of the individual assets. MCmarket allows one to select between the normal, t and skewed generalized t (sgt) distribution. Where the sgt is the most general and thereby nests nests a number of alternative distributions.

The first part of this step comes down to deciding if you want returns to be fat or thin tailed. If you want thin tailed returns then use the normal or t distribution with many degrees of freedom. If you want fat tails then use the t-distribution with few degrees of freedom or the sgt distribution. Finally if you want to tinker with the skewness and/or kurtosis of the distributions then use the sgt.

The second part involves deciding how to the respective distribution parameters across the assets. For example do you want all assets to have the same mean and sd? or do you want them to have the same mean and different sd, or different means and sd ect.. This is up to the user and depends on the purpose of doing the Monte Carlo.

I will demonstrate how to do this using MC_market

For simplicity this example will use the normal distribution.

# Assigning mean = 10 and sd = 100 to all assets
m1 <- sim_market(
    corr_4,
    k = 1000,
    # selecting the pdf for the assets
    marginal_dist = "norm",
    # selecting parameters
    marginal_dist_model = list(mu = -10, sd = 20)) # single arguments are assigned to all assets

m1 %>% group_by(Asset) %>% 
    mutate(mean = mean(Return), sd = sd(Return))

Time-Series Properties

The final step is to decide on how to assign time-series properties across the assets. MCmarket has a function called sim_garch that allows users to induce time series properties into a vector of random numbers by setting the parameters of a ARMA(1,1) + APARCH(1,1) model. The sim_markets function then utilises this function via is ts_model argument.
